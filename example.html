Spacing Rules

  • • Minimum interior padding for controls: 12px (p-3)
  • • Card content: p-4 (16px)
  • • Card headers: p-3 pb-2 (12px, 8px bottom)
  • • Inter-chip gap: 8-12px (gap-2 or gap-3)
  • • No zero-gap clusters

Accessibility

Contrast Requirements

  • AA contrast on all text (4.5:1)
  • Status tints maintain AA contrast
  • Brand colors meet accessibility standards

Interactive Elements

  • Visible focus ring on all controls
  • Keyboard navigation order logical
  • Touch targets ≥40×40px

Copy Rules

✅ Do

  • • Use sentence case everywhere
  • • Write action-oriented button labels
  • • Keep helper text to one sentence
  • • Prefer numerals and units (30d, %, $)
  • • Be concise and neutral

❌ Don't

  • • Use title case or all caps
  • • Write passive or vague labels
  • • Use filler words or jargon
  • • Use the word "ensure"
  • • Write long explanations

Examples

Good

  • • "Connect wallet"
  • • "Deploy strategy"
  • • "30d return: +3.4%"
  • • "Enter strategy name"

Avoid

  • • "Connect Your Wallet"
  • • "Click to deploy strategy"
  • • "Thirty day return is positive"
  • • "Please ensure you enter a strategy name"

PHAROS Design System Integration

Automated Workflow

This PHAROS design system is automatically generated from your Figma file. When you update designs in Figma, the system automatically extracts new tokens and regenerates the CSS.

Update PHAROS Design System:
npm run pharos # Extract tokens + generate CSS
npm run pharos-tokens # Extract tokens from Figma
npm run pharos-css # Generate CSS from tokens
npm run watch-figma # Watch for Figma changes

The system includes a complete component library, Tailwind CSS integration, and automated CI/CD pipeline for seamless design-to-development workflow.
