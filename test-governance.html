Governance & Quality Controls

Versioning
  • Minor: new components/props; no breaking changes.
  • Patch: bug fixes, docs copy updates.
  • Major: breaking tokens/props or removed components. Provide a migration note.
Deprecation policy
  • Mark deprecated in docs with a badge and removal target.
  • Keep for ≥1 minor release with a migration path.
  • Remove in the next major, note in CHANGELOG.
PR checklist
  • Uses semantic tokens; no raw hex; spacing ∈ {`{4,8,12,16,24,32}`}
  • Only shadow-pharos or shadow-pharos-sm; single borders
  • Focus visible on all interactive elements
  • Docs specimen is static (no popovers); includes state grid where applicable
  • Snapshots updated (npm run test:update) and reviewed
  • CHANGELOG entry added

Token Validation

Required Token Keys
Global: brand, brand50, spacing, radius, shadow
Semantic: status colors, surface colors
Validation: npm run lint:tokens
Lint Guards
Spacing: Only {4,8,12,16,24,32} allowed
Shadows: Only pharos/pharos-sm allowed
Colors: No raw hex outside tokens
Command: npm run lint:tokens

Quality Standards

Token Compliance: All colors, spacing, and shadows must use semantic tokens
Accessibility: Focus rings visible, proper contrast ratios, keyboard navigation
Responsive Design: Components work across breakpoints with proper mobile behavior
Performance: Minimal bundle impact, efficient rendering patterns
Testing: Visual regression tests pass, component behavior verified
Documentation: Clear examples, state grids, usage guidance
